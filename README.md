## English

### Overview

LogPurifier is a **task-agnostic** log cleaning method: using **bidirectional co-occurrence
dependency scores** between log templates, it applies Mean-Shift clustering to automatically
identify and remove *free-standing messages* (independently inserted noise weakly related to
execution behavior, e.g. periodic heartbeats), so downstream analysis sees sequences closer
to the system's functional behavior.

> The paper is a Registered Report with only method and experiment plan — no official code
> or results. This repo reproduces from the paper's text.

### Scope

- ✅ Core algorithm (dependency score + Mean-Shift segmentation + cleaning)
- ✅ Anomaly detection evaluation (IM + OC-SVM, extended with PCA/IForest/LogCluster)
- ❌ Model inference (MI) task and LMM statistical analysis (out of scope)

### Implementation notes & deviations

- **OC-SVM**: the paper describes OC-SVM as semi-supervised (normal data only) and adopts the
  Loglizer toolkit. Loglizer does not ship an OC-SVM (its `SVM` is a supervised `LinearSVC`),
  so we implement OC-SVM with `sklearn.svm.OneClassSVM` to match the described semantics.
- **IM features**: IM's invariant search scales poorly with event dimension on BGL time-window
  data, so features use loglizer's native OOV to reduce dimensionality. IM (designed for
  HDFS session windows) often mines few effective invariants under BGL fixed time windows; we
  report its metrics as-is.

### 概述

### 实现说明与偏差

- **OC-SVM**：论文将 OC-SVM 描述为半监督（只用正常数据），并采用 Loglizer 工具包。Loglizer
  未提供 OC-SVM（其 `SVM` 为监督式 `LinearSVC`），故本复现用 `sklearn.svm.OneClassSVM`
  实现以符合论文所述语义。
- **IM 特征**：IM 的不变量搜索在 BGL 时间窗口数据上随事件维度增长扩展性差，故特征用 loglizer
  原生 OOV 限维。IM（原为 HDFS session 窗口设计）在 BGL 固定时间窗口下常挖不出多少有效不变量，
  其指标如实报告。
